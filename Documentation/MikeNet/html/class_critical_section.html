Detailed Description

Critical section class with Enter and Leave methods.

Remarks:
Michael Pryor, 6/28/2010.

Wraps around the CRITICAL_SECTION object.

When reading or writing to protected section, use Enter() and when done use Leave().

Remember to use Leave() or deadlock will occur.

When debug is enabled (_DEBUG is defined, done automatically by visual studio) additional checks will be performed to help identify when:

  • A thread uses Leave() without first taking control using Enter().
  • A CriticalSeciton object is cleaned up while in use by another thread.
