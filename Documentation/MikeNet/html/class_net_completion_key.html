Detailed Description

Completion key used by the completion port to identify instances, sockets and other objects.

Remarks:
Michael Pryor, 6/28/2010.

When queued events complete they are often passed to the completion port. If they are, they must be identified. To identify them the completion port is given a NetCompletionKey object and from this it can retrieve the entity that initiated the event and pass it relevant information about the event now that it has completed.
