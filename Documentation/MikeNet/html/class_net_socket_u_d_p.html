Detailed Description

Socket used for UDP data transfer.

Remarks:
Michael Pryor, 3/28/2010.

This class provides functionality specific to the UDP protocol.

This class is not thread safe. Send and RawSend are thread safe.

bool NetSocketUDP::Recv (  )  [virtual]

Starts a receive operation via TCP.

This method calls the winsock method WSARecvFrom.

If WSARecvFrom is successful the result of the operation (which will probably not complete instantly) will be passed to the completion port. Upon completion NetSocket::recvBuffer will be filled with received data and NetSocket::recvAddr will be filled with the address that the packet came from, which can be different to the address we are connected to.

If WSARecvFrom is unsuccessful the operation will not complete so the completion port will receive no notification. This means that we must manually set the overlapped event in the case of initial failure.

Warning:
Receive operations will fail when the thread exits, so if you start a receive operation and the thread that started it completes then the operation will fail and no data will be received.
Returns:
true if an error occurred and data was not properly received, false if no error occurred.
