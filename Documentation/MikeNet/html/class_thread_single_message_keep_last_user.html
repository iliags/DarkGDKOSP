StoreVector<ThreadMessageItem> ThreadSingleMessageKeepLastUser::lastMessage [private]

Local store of the last message that was sent to the encryption thread.

We use a local store, instead of a global store stored by ThreadSingleMessageKeepLast so that each individual packet can poll on its own last encryption message, rather than polling on the last encryption message sent by any packet.

A vector is needed because there must be one message per thread.

size_t ThreadSingleMessageKeepLastUser::threadsUsageCount [static, private]

Number of packets using the encryption threads.

Usage begins when Encrypt or Decrypt is first used, and ends when the packet is destroyed.

Access is controlled using Packet::threads critical section.
