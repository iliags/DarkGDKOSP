Detailed Description

TCP mode where each packet has a prefix indicating the size of the remaining data.

Remarks:
Michael Pryor, 6/28/2010.

Packets sent have a prefix added which indicates the size of the packet (excluding the prefix). Packets received must have this prefix.

This TCP mode is the most flexible and useful and is recommended for most purposes.

The prefix is 32 bits for the 32 bit version of this plugin and 64 bits for the 64 bit version.

This class is thread safe.
