Detailed Description

simple version of clMultithread class which avoids the need for threadID parameters

Remarks:
Michael Pryor, 6/16/2010.

This class is designed to maintain the high performance that clMultithread maintains but avoid the need for threadIDs which can reduce flexibility. Usage is the same as the clMultithread class except for:

  • EnterRead may only be used once prior to a LeaveRead call per thread (not doing so causes deadlock)
  • EnterWrite may only be used once prior to a LeaveWrite call per thread (not doing so causes deadlock)
  • One thread cannot have read and write control (attempting to do so causes deadlock)
  • Small reduction in performance

void clMultithreadSimple::EnterRead (  )  const

Take read control of object, write control will not be possible until LeaveRead is used
EnterRead can be used multiple times before LeaveRead but is is essential that the same number of LeaveRead's are used to prevent deadlock
In contrast to clStatus, if the calling thread has read control, write control will not be possible!

void clMultithreadSimple::EnterWrite (  ) 

Take write control of object, write control cannot be taken unless no thread has read or write control (including the thread calling this method). The method will block until write control can be taken.
It is essential that a call to EnterWrite is counteracted by a call to LeaveWrite.

bool clMultithreadSimple::IsWriteAvailable (  )  const
Returns:
true if EnterWrite will not block, note that between this call and a subsequent EnterWrite call it is possible that another thread will take write control so you cannot rely on this behavior.
void clMultithreadSimple::LeaveRead (  )  const

Release read control of object, write control will be possible when all EnterRead operations are reversed by the same number of LeaveRead operations.
In contrast to clStatus, if the calling thread has read control, write control will not be possible! It is essential that this method is not used unless the thread already has read control. Note that there are no safe guards against this so be careful!

void clMultithreadSimple::LeaveWrite (  ) 

Release write control of object, write control must not be released unless the calling thread already has it. Note that there are no safeguards against this so be careful!
It is essential that a call to EnterWrite is counteracted by a call to LeaveWrite.
