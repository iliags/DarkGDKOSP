enum clNetworkingUtility::ConnectionStatus

Describes the stage that a connection is in

Enumerator:
REFUSED 

Connection was refused.
Value of -2.

CONNECTION_ERROR 

Error occurred whilst connecting.
Value of -1.

TIMED_OUT 

Connection timed out, i.e. did not complete within required amount of time.
Value of 0.

NOT_CONNECTED 

Not connected at all i.e. inactive.
Value of 0.

CONNECTED 

Fully connected i.e. active.
Value of 1.

STILL_CONNECTING 

Connection attempt is still in progress.
Value of 2.

NO_SEND 

The graceful disconnection process is in progress and data may no longer be sent via TCP but may still be received. Other protocols are unaffected. This status is received when the connection has been shut down; see graceful disconnecting for more information.
Value of 2.

NO_RECV 

The graceful disconnection process is in progress and data may no longer be received via TCP but may still be sent. Other protocols are unaffected. This status is received when the connection has been shut down by the other participant; see graceful disconnecting for more information.
Value of 3.

NO_SEND_RECV 

The graceful disconnection process is in progress and data may no longer be sent or received via TCP Other protocols are unaffected. This status is received when the connection has been shut down by both participants but there are packets left in the receive queue; see graceful disconnecting for more information.
Value of 4.

CONNECTING 

The connection process is in progress Value of 5.

CONNECTED_AC 

The connection has completed but confirmation is needed before the connection becomes fully active.
Value of 6.

DISCONNECTING 

The connection is still active but a request has been made for it to be terminated.
Value of 7.

enum clNetworkingUtility::Error

Values used to indicate error

Enumerator:
NET_ERROR 

Attempted a networking operation but failed, used where the return value indicates error and is signed. This is the most common error return value.
Value of -1.

UNSIGNED_NET_ERROR 

Attempted a networking operation but failed, used where the return value indicates error and is unsigned.
Value of 0.

enum clNetworkingUtility::SendStatus

Describes the result of a send operation.

Enumerator:
SEND_COMPLETED 

The send operation completed successfully and the data has now been sent.
Value of 0.

SEND_IN_PROGRESS 

The send operation was started, but has not yet completed.
Value of 1.

SEND_FAILED 

The send operation failed, but not catastrophically, so the entity does not need to be killed.
Value of -1.

SEND_FAILED_KILL 

The send operation failed catastrophically, so the entity should be killed. Value of -2.
