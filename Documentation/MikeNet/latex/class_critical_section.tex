\subsection{Detailed Description}
Critical section class with Enter and Leave methods. \begin{DoxyRemark}{Remarks}
Michael Pryor, 6/28/2010.
\end{DoxyRemark}
Wraps around the CRITICAL\_\-SECTION object. \par
\par


When reading or writing to protected section, use \hyperlink{class_critical_section_a509be5342b2f07ae06e1af5d9e08b871}{Enter()} and when done use \hyperlink{class_critical_section_a6fa0931385f066492f31fc98e3fca657}{Leave()}. \par
\par


Remember to use \hyperlink{class_critical_section_a6fa0931385f066492f31fc98e3fca657}{Leave()} or deadlock will occur.\par
\par


When debug is enabled (\_\-DEBUG is defined, done automatically by visual studio) additional checks will be performed to help identify when:
\begin{DoxyItemize}
\item A thread uses \hyperlink{class_critical_section_a6fa0931385f066492f31fc98e3fca657}{Leave()} without first taking control using \hyperlink{class_critical_section_a509be5342b2f07ae06e1af5d9e08b871}{Enter()}.
\item A CriticalSeciton object is cleaned up while in use by another thread. 
\end{DoxyItemize}
