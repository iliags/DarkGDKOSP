DarkGDK Open Source Project

Super 3D Game Engine - A Powerful Game Engine capable of producing unique 3D Computer Games based on combined game mechanics from popular computer game genres/sub-genres: FPS, RPG, RTS, Puzzle, Arcade, and many others; viewable from a variety of camera perspectives. The Engine should include scalable Multiplayer support for Single Player up to Massive Multiplayer Games. MOTIVATION: Provide a Free, Open Source, High Performance, Modular Game Engine that will allow DGDK Game Developers to dive immediately into the Game Content Creation (Media & Logic) and experiment with various game mechanics without the need to alter source code. if(a feature is not readily available, a Plug-in architecture makes addition new features and modifications a snap!

Getting Started: The DOSP Restructure

DarkGDK Open Source Project Restructure is an effort to 1) Reorganize the project in a more intuitive fashion; 2) Stablize Open Source distribution via SVN for all contributors; 3) Minimize Version conflicts. For self-containment, the restructure will result in separate copies of DGDK and other 3rd Party Libraries from the standard DGDK default directories. This structure is based on the Directory Structure recommendations from the Core Team Members.

  1. DOSP – Root Directory

    1. Documentation – 3rd Party Libraries, Plugins, and Tool Documentation.

    2. Include – 3rd Party C++ Header and Source Files.

    3. Lib – 3rd Party Static Library Files.

    4. Dll – 3rd Party Dynamic Link Libraries.

    5. Projects – DOSP Projects.

    6. Tools – Utility Executables.

    7. Content – S3GP Application/Game Media, Data, and Logic Files. This sub directory is located in the directory that contains the S3GP.exe (which is currently generated and located in Projects\S3GP\S3GP\Debug\) . This directory contains the cached Content Files for Application / Game. Note: S3GP Host Server will provide a centralized respository for all Content Files downloaded S3GP upon requirement.

In this stage of development, the project is compiled in debug mode and should generate object and executable files in Projects\S3GP\S3GP\Debug\

The Debug/Release Directory represents the Root Directory for S3GP.exe and associated files.

The Content Directory (currently located in the Debug Folder) represents an Application/Game's cached content {media, auxillary data, markup & script} files for use by S3GP.

VC++ IDE requires you specify the path and directorie of *lib files in order to properly locate the Lib Directories for DOSP. From within VC++ add the following entry:

Tools → Options... → Project and Solutions → VC++ Directories → Show directories for: Library files → [Absolute Path of DOSP]\DOSP\Lib\vs9

Known Bugs:

  1. Problem: SOMESTDLIB

    Details: Something isn't supported by SOMESTDLIB use /NoDefaultLib when compiling under 'Debug'

    Fix: Go to Project->Properties->Configuration properties->C/C++->Code generation->Runtime library Change the runtime library from Multi-threaded Debug (MTd) to Multi-threaded(MT).

  2. Problem: Compiling Box2D

    Details: Linking... libcpmtd.lib(stdthrow.obj) : error LNK2019: unresolved external symbol __CrtDbgReportW referenced in function "void __cdecl std::_Debug_message(wchar_t const *,wchar_t const *,unsigned int)" (?_Debug_message@std@@YAXPB_W0I@Z) Debug\S3GP.exe : fatal error LNK1120: 1 unresolved externals

    Fix: Compile Lib in Release mode Reference: TGC Forum Thread

  3. Problem: IIrrXMLReader

    Details: LNK2019: unresolved external symbol with stuff about irr::io::IIrrXMLReader

    Fix: *Optional but recommended is to make a new filter called xmlplugin or something like that* Right click on either 'Source Files' or the filter you made for the xmlplugin as mentioned previously :) Add->Existing Item->Browse to the 'include' folder (up 3 levels from S3GP project)->Browse to the 'xmlplugin' folder->Select all of the files and then click 'add'

  4. Problem: Audio Plugin

    Details: "This application has failed to start because irrKlang.dll was not found. RE-installing the application may fix this problem."

    Fix: Copy the irrKlang.dll and ikpMP3.dll from DarkGDK\Media\dlls into the folder that S3GP.exe is in.
